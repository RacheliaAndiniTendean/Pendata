1. Pendahuluan#

Pada tugas ini dilakukan analisis perbandingan antara algoritma klasifikasi Decision Tree dan Random Forest menggunakan aplikasi KNIME serta Python Script Legacy.

Dataset yang digunakan adalah Online Shoppers Purchasing Intention Dataset. Dataset ini berisi data aktivitas pengunjung pada website e-commerce, seperti jumlah halaman yang dibuka, lama kunjungan, browser yang digunakan, jenis visitor, traffic website, hingga informasi apakah pengunjung melakukan transaksi pembelian atau tidak.

Target klasifikasi yang digunakan adalah:

Revenue

Keterangan:

TRUE  = pengunjung melakukan pembelian
FALSE = pengunjung tidak melakukan pembelian

Tujuan penelitian ini adalah:

  1. Membuat model klasifikasi menggunakan Decision Tree

  2. Membuat model klasifikasi menggunakan Random Forest

  3. Membandingkan performa kedua algoritma

  4. Mengevaluasi hasil prediksi menggunakan confusion matrix

2. Deskripsi Dataset#

Dataset terdiri dari atribut numerik dan atribut kategorikal.

2.1 Atribut Numerik#

Atribut numerik yang digunakan antara lain:

Administrative
Administrative_Duration
Informational
Informational_Duration
ProductRelated
ProductRelated_Duration
BounceRates
ExitRates
PageValues
SpecialDay

a. Administrative#

Jumlah halaman administratif yang dibuka pengunjung.

b. Administrative_Duration#

Total waktu yang dihabiskan pada halaman administratif.

c. Informational#

Jumlah halaman informasi yang dibuka.

d. Informational_Duration#

Total waktu pada halaman informasi.

e. ProductRelated#

Jumlah halaman produk yang dibuka pengunjung.

f. ProductRelated_Duration#

Durasi waktu yang dihabiskan pada halaman produk.

Contoh:

ProductRelated = 10
ProductRelated_Duration = 627.5

Artinya pengunjung membuka 10 halaman produk dan menghabiskan waktu sekitar 627.5 detik.

g. BounceRates#

Persentase pengunjung yang langsung keluar dari situs tanpa melakukan interaksi lain.

h. ExitRates#

Persentase halaman yang menjadi halaman terakhir sebelum pengunjung keluar.

i. PageValues#

Nilai rata-rata halaman sebelum terjadi transaksi pembelian.

j. SpecialDay#

Menunjukkan kedekatan kunjungan dengan hari spesial tertentu seperti:

  • Valentine

  • Hari Ibu

  • Hari Raya

3. Workflow KNIME#

Pada workflow ini digunakan beberapa node utama untuk membangun model klasifikasi menggunakan algoritma Decision Tree dan Random Forest. Setiap node memiliki fungsi yang berbeda, mulai dari membaca dataset, membagi data, melakukan training model, melakukan testing, sampai mengevaluasi hasil prediksi.

Workflow yang digunakan secara umum adalah:

CSV Reader
↓
Table Partitioner
├── Python Script Training Decision Tree
├── Python Script Testing Decision Tree => Scorer Decision Tree
├── Python Script Training Random Forest
└── Python Script Testing Random Forest => Scorer Random Forest

4. Penjelasan Node pada Workflow KNIME#

Pada workflow ini digunakan beberapa node utama untuk membangun model klasifikasi menggunakan algoritma Decision Tree dan Random Forest. Setiap node memiliki fungsi yang berbeda dalam proses preprocessing, training, testing, hingga evaluasi model.

4.1 CSV Reader#

Node CSV Reader digunakan untuk membaca dataset Online Shoppers Purchasing Intention yang berbentuk file .csv.

Pada tahap ini, seluruh data dimasukkan ke dalam workflow KNIME agar bisa diproses pada tahap berikutnya.

Beberapa hal yang perlu diperhatikan pada node ini adalah:

  • separator data

  • tipe data kolom

  • header dataset

  • encoding file

  • apakah kolom numerik dan kategorikal sudah terbaca dengan benar

Dataset yang berhasil dibaca akan menghasilkan tabel berisi atribut numerik dan kategorikal.

4.2 Table Partitioner#

Node Table Partitioner digunakan untuk membagi dataset menjadi data training dan data testing.

Pada workflow ini, data dibagi menjadi:

\[ 80\% = data\ training \]
\[ 20\% = data\ testing \]

Data training digunakan untuk melatih model, sedangkan data testing digunakan untuk menguji hasil prediksi model.

Output dari node ini terdiri dari dua bagian:

First Partition  = data training
Second Partition = data testing

Data training akan masuk ke node Python Script untuk proses training model. Data testing akan masuk ke node Python Script untuk proses prediksi.

4.3 Node Decision Tree#

Bagian Decision Tree digunakan untuk membangun model klasifikasi dengan algoritma Decision Tree.

Pada bagian ini terdapat tiga node utama:

  1. Python Script Training Decision Tree

  2. Python Script Testing Decision Tree

  3. Scorer Decision Tree

4.3.1 Python Script Training Decision Tree#

Node Python Script Training Decision Tree digunakan untuk melatih model Decision Tree menggunakan data training dari Table Partitioner.

Pada node ini, data training dari KNIME diambil ke Python, kemudian dilakukan preprocessing, pembuatan model, training, dan penyimpanan model menggunakan pickle.

Kode lengkap yang digunakan:

import pandas as pd
import os
import pickle
import numpy as np

from sklearn.tree import DecisionTreeClassifier
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder
from sklearn.impute import SimpleImputer

# Ambil data training
df = input_table_1.copy()

# Target
target_col = "Revenue"

# Bersihkan data
df = df.replace("?", np.nan)

for col in df.select_dtypes(include=["object"]).columns:
    df[col] = df[col].astype(str).str.strip()

# Kolom numerik
numeric_columns = [
    "Administrative",
    "Administrative_Duration",
    "Informational",
    "Informational_Duration",
    "ProductRelated",
    "ProductRelated_Duration",
    "BounceRates",
    "ExitRates",
    "PageValues",
    "SpecialDay"
]

for col in numeric_columns:
    if col in df.columns:
        df[col] = pd.to_numeric(df[col], errors="coerce")

# Kolom kategorikal
categorical_columns = [
    "Month",
    "OperatingSystems",
    "Browser",
    "Region",
    "TrafficType",
    "VisitorType",
    "Weekend"
]

for col in categorical_columns:
    if col in df.columns:
        df[col] = df[col].astype(str)

df = df.dropna(subset=[target_col])

# Pisahkan fitur dan target
X_train = df.drop(columns=[target_col])
y_train = df[target_col].astype(str)

# Fitur numerik dan kategorikal
numeric_cols = [col for col in numeric_columns if col in X_train.columns]
categorical_cols = [col for col in categorical_columns if col in X_train.columns]

try:
    encoder = OneHotEncoder(handle_unknown="ignore", sparse_output=False)
except TypeError:
    encoder = OneHotEncoder(handle_unknown="ignore", sparse=False)

numeric_transformer = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="median"))
])

categorical_transformer = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="most_frequent")),
    ("encoder", encoder)
])

preprocessor = ColumnTransformer(
    transformers=[
        ("num", numeric_transformer, numeric_cols),
        ("cat", categorical_transformer, categorical_cols)
    ]
)

# Model Decision Tree
decision_tree = DecisionTreeClassifier(
    criterion="gini",
    min_samples_leaf=2,
    random_state=42
)

model_pipeline = Pipeline(steps=[
    ("preprocessing", preprocessor),
    ("decision_tree", decision_tree)
])

# Training
model_pipeline.fit(X_train, y_train)

# Simpan model
model_folder = os.path.join(os.path.expanduser("~"), "knime_model")
os.makedirs(model_folder, exist_ok=True)

model_path = os.path.join(model_folder, "decision_tree_revenue_model.pkl")

with open(model_path, "wb") as file:
    pickle.dump(model_pipeline, file)

# Output
output_table_1 = pd.DataFrame({
    "Status": ["Decision Tree berhasil dibuat dan disimpan"],
    "Model Path": [model_path],
    "Jumlah Data Training": [len(df)],
    "Target": [target_col],
    "Jumlah Fitur": [X_train.shape[1]]
})

from sklearn.ensemble import RandomForestClassifier
from sklearn.pipeline import Pipeline

# Ambil data training
df = input_table_1.copy()

# Target
target_col = "Revenue"

df = df.replace("?", np.nan)

for col in df.select_dtypes(include=["object"]).columns:

# Kolom numerik
numeric_columns = [

    "ProductRelated",

    "BounceRates",

# Kolom kategorikal

])

categorical_transformer = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="most_frequent")),
    ("encoder", encoder)
])

preprocessor = ColumnTransformer(
    transformers=[
        ("num", numeric_transformer, numeric_cols),
        ("cat", categorical_transformer, categorical_cols)
    ]
)

# Model Random Forest

    n_estimators=50,
    min_samples_leaf=1,
    max_depth=None,
    random_state=42,
    n_jobs=-1
)

    ("preprocessing", preprocessor),
    ("random_forest", random_forest)
])

5.1 Confusion Matrix#

Confusion matrix digunakan untuk melihat jumlah prediksi benar dan salah pada masing-masing kelas.

Pada dataset ini target Revenue memiliki dua kelas:

FALSE = tidak melakukan pembelian
TRUE  = melakukan pembelian

Bentuk umum confusion matrix:

Actual FALSE => Predicted FALSE = TN
Actual FALSE => Predicted TRUE  = FP
Actual TRUE  => Predicted FALSE = FN
Actual TRUE  => Predicted TRUE  = TP

Keterangan:

TN = True Negative
FP = False Positive
FN = False Negative
TP = True Positive

a. True Negative#

True Negative adalah data dengan nilai asli FALSE dan diprediksi FALSE.

Artinya model berhasil memprediksi bahwa pengunjung tidak melakukan pembelian.

b. False Positive#

False Positive adalah data dengan nilai asli FALSE, tetapi diprediksi TRUE.

Artinya model salah memprediksi bahwa pengunjung melakukan pembelian.

c. False Negative#

False Negative adalah data dengan nilai asli TRUE, tetapi diprediksi FALSE.

Artinya model gagal mendeteksi pengunjung yang sebenarnya melakukan pembelian.

d. True Positive#

True Positive adalah data dengan nilai asli TRUE dan diprediksi TRUE.

Artinya model berhasil memprediksi bahwa pengunjung melakukan pembelian.

5.2 Accuracy#

Accuracy digunakan untuk menghitung seberapa banyak prediksi model yang benar dibandingkan seluruh data testing.

Rumus accuracy:

\[ Accuracy = \frac{TP + TN}{TP + TN + FP + FN} \]

Semakin tinggi nilai accuracy maka semakin baik performa model secara keseluruhan.

5.3 Precision#

Precision digunakan untuk melihat ketepatan model saat memprediksi kelas positif.

Rumus precision:

\[ Precision = \frac{TP}{TP + FP} \]

Precision tinggi menunjukkan bahwa hasil prediksi positif model cukup akurat.

5.4 Recall#

Recall digunakan untuk melihat kemampuan model dalam menemukan data positif.

Rumus recall:

\[ Recall = \frac{TP}{TP + FN} \]

Semakin tinggi recall maka model semakin baik dalam mendeteksi data positif.

5.5 F1-Score#

F1-Score digunakan untuk melihat keseimbangan antara precision dan recall.

Rumus F1-Score:

\[ F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall} \]

Nilai F1-Score biasanya digunakan ketika ingin melihat performa model secara lebih seimbang.

Hal ini menunjukkan bahwa penggunaan banyak decision tree mampu membantu model menghasilkan prediksi yang lebih stabil dan lebih akurat dalam memprediksi apakah pengunjung melakukan pembelian atau tidak.
