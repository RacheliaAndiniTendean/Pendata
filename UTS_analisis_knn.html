UTS-Analisa Data Kesuburan Tanah#

Pada tahap ini dilakukan evaluasi terhadap model klasifikasi menggunakan algoritma K-Nearest Neighbor (KNN). Evaluasi dilakukan menggunakan beberapa metrik yaitu Accuracy, Precision, Recall, dan F1-Score.

1. Data Understanding#

Dataset yang digunakan berisi data kondisi tanah dengan label:

  • Subur (positif)

  • Tidak Subur (negatif)

Fitur meliputi:

  • Fitur numerik: pH, N Total, P Tersedia, K Tersedia, C Organik, KTK, Kejenuhan Basa, Kadar Air, Bulk Density

  • Fitur kategorikal: Tekstur Tanah

  • Label: Subur / Tidak Subur

2. Data Preprocessing#

Tahap preprocessing dilakukan untuk menyiapkan data agar dapat digunakan dalam algoritma KNN.

2.1 Missing Value Imputation#

Digunakan untuk mengatasi data yang kosong (missing).

  • Numerik → diisi dengan Mean

  • Kategorikal → diisi dengan Most Frequent (modus)

Tujuan:

  • Menghindari error saat perhitungan

  • Menjaga distribusi data

2.2 Encoding Data Kategorikal#

Fitur kategorikal seperti Tekstur Tanah diubah menjadi numerik menggunakan One-Hot Encoding (One to Many).

Contoh:

  • Pasir → (1,0,0)

  • Lempung → (0,1,0)

  • Liat → (0,0,1)

Tujuan:

  • Agar bisa dihitung jaraknya oleh KNN

2.3 Normalisasi Data#

Dilakukan menggunakan Min-Max Normalization:

\[ x' = \frac{x - min}{max - min} \]

Tujuan:

  • Menyamakan skala fitur

  • Mencegah fitur dengan nilai besar mendominasi jarak

2.4 Pembagian Data (Partitioning)#

Data dibagi menjadi:

  • Training = 70% (1400 data)

  • Testing = 30% (600 data)

Menggunakan:

  • Stratified Sampling → menjaga proporsi kelas tetap seimbang

3. Algoritma K-Nearest Neighbor (KNN)#

KNN adalah algoritma klasifikasi berbasis jarak.

3.1 Konsep KNN#

  • Untuk setiap data testing:

    • Hitung jarak ke semua data training

    • Ambil K tetangga terdekat

    • Tentukan kelas berdasarkan voting mayoritas

3.2 Perhitungan Jarak (Euclidean Distance)#

\[ d = \sqrt{\sum (x_i - y_i)^2} \]

3.3 Parameter K#

Nilai K yang digunakan:

  • K = 3

  • K = 5 (perbandingan)

Semakin kecil K:

  • lebih sensitif

Semakin besar K:

  • lebih stabil tapi bisa menurunkan akurasi

4. Confusion Matrix#

5. Metrik Evaluasi#

Komponen dari confusion matrix:

  • True Positive (TP) → Subur diprediksi Subur

  • True Negative (TN) → Tidak Subur diprediksi Tidak Subur

  • False Positive (FP) → Tidak Subur diprediksi Subur

  • False Negative (FN) → Subur diprediksi Tidak Subur

5.1 Accuracy#

\[ Accuracy = \frac{TP + TN}{TP + TN + FP + FN} \]

5.2 Precision#

\[ Precision = \frac{TP}{TP + FP} \]

5.3 Recall#

\[ Recall = \frac{TP}{TP + FN} \]

5.4 F1-Score#

\[ F1 = \frac{2 \times Precision \times Recall}{Precision + Recall} \]

6. Hasil Evaluasi#

Hasil:

  • Accuracy = 54.7%

  • Precision ≈ 51%

  • Recall ≈ 55%

  • F1-score ≈ 53%

7. Analisis#

Dilihat dari confusion matrix, jumlah prediksi yang salah masih cukup tinggi. Hal ini terlihat dari nilai False Positive (146) dan False Negative (126) yang relatif besar. Artinya:

  • Model sering mengklasifikasikan data Tidak Subur sebagai Subur

  • Model juga cukup sering gagal mendeteksi data Subur

Nilai precision yang lebih rendah (~51%) menunjukkan bahwa prediksi kelas Subur masih banyak yang tidak tepat. Sementara itu, recall (~55%) menunjukkan bahwa model cukup mampu mendeteksi data Subur, namun belum optimal.

Nilai F1-score (~53%) mengindikasikan bahwa keseimbangan antara precision dan recall masih berada pada tingkat sedang, sehingga performa keseluruhan model belum stabil.

8. Kesimpulan#

Model K-Nearest Neighbor (KNN) yang digunakan pada penelitian ini mampu melakukan klasifikasi data kesuburan tanah, namun performanya masih belum optimal.

Hal ini ditunjukkan oleh nilai accuracy yang hanya mencapai 54.7% serta nilai precision, recall, dan F1-score yang masih berada pada kisaran sedang. Dengan demikian, model masih sering melakukan kesalahan dalam membedakan antara kelas Subur dan Tidak Subur.
